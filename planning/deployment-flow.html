Network Architecture Details

  • VPC CIDR: 10.0.0.0/16 (65,536 IP addresses)
  • Public Subnets: 10.0.1.0/24, 10.0.2.0/24 (ALB and NAT Gateways)
  • Private Subnets: 10.0.10.0/24, 10.0.11.0/24 (ECS Tasks)
  • Security Groups: Restrictive ingress/egress rules
  • High Availability: Multi-AZ deployment for fault tolerance

Blue-Green Deployment Strategy

Zero-downtime deployment process using ECS blue-green deployment strategy with automatic rollback capabilities.
sequenceDiagram participant Dev as Developer participant GH as GitHub participant GA as GitHub Actions participant ECR as Amazon ECR participant ECS as Amazon ECS participant ALB as Load Balancer participant CW as CloudWatch participant User as End User Dev->>GH: Push code to main branch GH->>GA: Trigger deployment workflow GA->>GA: Run tests and build GA->>ECR: Push new container images GA->>ECS: Create new task definition ECS->>ECS: Start blue environment (new) ECS->>ECR: Pull new images ECS->>CW: Report startup metrics Note over ECS,ALB: Blue-Green Switch ECS->>ALB: Register new tasks (blue) ALB->>ALB: Health check new tasks ALB->>User: Route 10% traffic to blue CW->>CW: Monitor error rates alt If metrics healthy ALB->>User: Gradually shift to 100% blue ECS->>ECS: Terminate green tasks GA->>Dev: Deployment successful else If metrics unhealthy ALB->>User: Route 100% back to green ECS->>ECS: Terminate blue tasks GA->>Dev: Deployment failed, rolled back CW->>Dev: Send alert notification end
Deployment Stages:
  1. New version deployed to blue environment
  2. Health checks validate blue environment
  3. Canary deployment (10% traffic)
  4. Monitor metrics for 5 minutes
  5. Gradual traffic shift (25%, 50%, 100%)
  6. Keep green environment for 15 minutes (quick rollback)
  7. Terminate green environment

Rollback Decision Tree:
  • < 30 seconds: Use blue-green switch (if green still running)
  • < 5 minutes: Revert ECS task definition
  • < 1 hour: Deploy previous ECR image
  • > 1 hour: Full Terraform state rollback

Quick Reference Commands

Common commands for managing the deployment pipeline and infrastructure.

# Initialize Terraform

terraform init


# Plan infrastructure changes

terraform plan -var-file=environments/dev.tfvars


# Apply infrastructure

terraform apply -var-file=environments/dev.tfvars


# Login to ECR

aws ecr get-login-password --region us-west-2 | docker login --username AWS --password-stdin [account].dkr.ecr.us-west-2.amazonaws.com


# Push to ECR

docker tag durable-code-backend:latest [account].dkr.ecr.us-west-2.amazonaws.com/durable-code-backend:latest

docker push [account].dkr.ecr.us-west-2.amazonaws.com/durable-code-backend:latest


# Update ECS service

aws ecs update-service --cluster durable-code-cluster --service backend-service --force-new-deployment


# View logs

aws logs tail /ecs/durable-code-backend --follow


# Emergency rollback

aws ecs update-service --cluster durable-code-cluster --service backend-service --task-definition backend-task:PREVIOUS_REVISION

Cost Optimization Tips:
  • Use Fargate Spot for development/staging (70% savings)
  • Single NAT Gateway for non-production (save $45/month)
  • Consider removing NAT entirely and use VPC endpoints
  • Implement auto-scaling to scale down during low traffic
  • Use CloudFront for static assets to reduce ALB load
  • Set up AWS Budget alerts at $50/month threshold
